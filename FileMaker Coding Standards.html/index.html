Indenting & White Space
Indent calculation code using hard tabs (no spaces) for better readability in the calculation dialog box.
When editing code within the calc dialog box you can use Option-Tab (Mac) or Control-Tab (Win) to insert tabs
No trailing white space after lines of code.

Fields: key fields
The primary key in each table is simply named "id" - Fields are identified by FileMaker inherently by the TableName::id nomenclature.
Primary keys should use a UUID value (by custom function or plugin) via auto-enter and do not use FileMaker's serial value option.
Using a UUID addresses any issues with importing and forgetting to modify any serial number - Just make sure to not perform auto-enter on import.
Foreign keys within any table are named using an underscore followed by the lowercase version of their Title case table name. (e.g id_table). Only foreign keys use the underscore.
The underscore within the field name distinguishes a foreign key from the camelCase field names

Boolean results
Any returned result in your code, which is boolean, is written using FileMaker's support for "True" and "False" instead of using return values of zero (0) and (1). This increases readability of code.

Within solution code this increases code readability and distinguishes literal values and escaped text from empty values.
